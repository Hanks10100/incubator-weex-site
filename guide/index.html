What is Weex?#

Weex is a framework for building high-performance mobile apps with modern web development experience.

Weex dedicate to enable developers to use modern web development experience to build both Android, iOS and web applications with a single codebase. In practice, you can use javascript and modern front-end frameworks to develop mobile apps after integrate the WeexSDK.

The structure of Weex is decoupled, the render engines are separate from the syntax layer. Weex doesn’t relay on any specific front-end frameworks, instead, both Vue.js and Rax can be performed on Weex.

Another primary goal of Weex is to keep up with modern development technologies and platform capabilities both on web and native. Productivity and performance can be coexisted in Weex. When writing Weex pages, you feel the same with writing web pages. When rendering Weex pages, you feel the same with rendering native pages.

Native Components#

In the example above, the <div> and the <text> is rendered into corresponding native views on mobile device, not HTMLElement.

Weex implemented the render engine in each end.

Under the scene, Weex is using the OEM (native) widgets.

因为实际生成的是原生组件，所以你并不能使用 HTML 中定义的标签，WeexSDK 内置了一套组件集，包含了常用的组件，可以覆盖到大部分使用场景。基于这些组件，你可以使用 Vue.js 的语法封装更上层的自定义组件。

Weex is not a shell of webview, all components is rendered into native views on mobile devices.

Weex has many build-in components, it can support the basic usage.

Except for the build-in components, you can also extend your own native components. But you have to both extend Android and extend iOS.

Native Modules#

Weex wrap many native utilities into modules. You can use weex.requireModule('xxx') to require them.

Weex also lets you reuse your existing Java, Swift, and ObjC code, and access native features and SDKs on iOS and Android.

Support Multiple Front-End Frameworks#

Test, build, debug, tools, 3td-party libs.

Front end is flourishing and productive.

Weex is not a front-end framework, it contains Android and iOS SDK. Actually, Weex is more like a platform than a framework. Modern front-end frameworks, iOS or Android extensions are able to integrate to Weex.

Vue.js 2.0 and Rax

Those frameworks are packaged into Weex SDK, you don’t need to require them manually.

upgrade.

Rax is a front-end framework with largely React-compatible APIs.

Weex isn’t binding with any specific front-end frameworks, the DOM operations will convert into render directives send to native render engine.

You can also extend the Weex SDK with your favorite front-end framework, Weex can bring the ability of rendering native components to it. But it’s complicate and tricky, you must know many details about our js-native bridge and native render engines. You can read Extend JS Framework to learn the details.

Create Your Own App#

The following steps assume you have already known the foundational knowledge of Node.js and npm. If you are not familiar with them, you can visit https://docs.npmjs.com/ to learn more about npm.

Weex provide a command line tool, the weex-toolkit, to help developers to get start easily. It can help you create a starter project, setup iOS and Android development environments, debug, install plugins and so on.

Currently, the weex-toolkit only supports to create Vue.js project. The rax-cli maybe helpful if you want to use Rax. Please visit Rax’s official website for more details.

Setup#

Make sure you have already installed the Node.js, and then install the weex-toolkit globally.

It will add the weex command to your global path. Run weex --help can see the manual of all its sub commands. The most useful command is weex create [project-name]:

After doing that, you’ll get a standard Weex + Vue.js project under the awesome-app folder.

Develop#

The next step is get into the path, install dependencies, and start.

It will start a web server on the 8080 port. You can open http://localhost:8080 to see the result rendered on the web. Besides, You can also run weex src/index.vue to preview the web render result in iframe and with a QR code generated on the right, which you can use the Weex playground app to scan it to see the render result on the mobile.

Same as Web. Be aware of the difference between Weex and web platform.
